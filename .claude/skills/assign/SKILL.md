# Assign — Decompose & Route Task

ใช้เมื่อ user ให้ task และ ARIA ต้องวิเคราะห์ว่าจะมอบหมายให้ agent ไหน

## Usage

```
/assign <task description>
```

---

## Execution Modes

| Mode | เมื่อไหร่ | วิธีรัน |
|------|----------|---------|
| **Sequential** | Agent B ต้องการผลจาก Agent A | รัน A → รอ → รัน B |
| **Parallel** | Agents ทำงาน independent | รัน A & B พร้อมกัน → `wait` |
| **Mixed** | BE ก่อน → FE+QA พร้อมกัน | Sequential round 1 → Parallel round 2 |

---

## Protocol

### Step 1: Analyze the task

ถาม 4 คำถาม:
1. มี server-side logic / API / DB / auth? → **Kai**
2. มี UI / component / styling / FE state? → **Nova**
3. ต้องการ testing / review / security? → **Sage** (เกือบทุก task)
4. มี dependency ระหว่าง agents? → กำหนด mode

---

### Step 2: Create execution plan

```markdown
## Execution Plan: [Task Title]

**Task ID:** TASK-[number]
**Mode:** sequential | parallel | mixed

### Breakdown

| Round | Agent | Task | Depends on | Mode |
|-------|-------|------|------------|------|
| 1 | Kai | [desc] | — | foreground |
| 2 | Nova | [desc] | Round 1 | foreground |
| 2 | Sage | [desc] | Round 1 | parallel with Nova |

### Definition of Done
- [ ] [criterion]
- [ ] QA approved
```

---

### Step 3: Get user approval (complex tasks only)

ถ้า complexity = **complex** หรือมี architecture decision → present plan ก่อน  
ถ้า **simple/medium** → proceed อัตโนมัติ

---

### Step 4: Execute — Write orchestration script

ARIA เขียน bash script แล้วรันผ่าน Bash tool:

```bash
#!/bin/bash
# Generated by ARIA for TASK-[id]
PROJECT_DIR="$(pwd)"
TOOLS_RW="Edit,Write,Read,Bash,Glob,Grep"
TOOLS_RO="Read,Bash,Glob,Grep"

# === Round 1: [agent A] ===
echo "▶ Round 1: [Agent A]..."
cd "$PROJECT_DIR" && claude -p "$(cat _shared/briefs/[agent-a]-[id].md)" \
  --allowed-tools "$TOOLS_RW"
echo "✅ Round 1 done"

# === Round 2: [Agent B] + [Agent C] พร้อมกัน ===
echo "▶ Round 2: [B] + [C] parallel..."
cd "$PROJECT_DIR" && claude -p "$(cat _shared/briefs/[agent-b]-[id].md)" \
  --allowed-tools "$TOOLS_RW" 2>&1 &
B_PID=$!

cd "$PROJECT_DIR" && claude -p "$(cat _shared/briefs/[agent-c]-[id].md)" \
  --allowed-tools "$TOOLS_RO" 2>&1 &
C_PID=$!

wait $B_PID && echo "✅ [B] done"
wait $C_PID && echo "✅ [C] done"
```

**3 สิ่งที่ script ต้องมีเสมอ:**
1. `--allowed-tools` — ห้ามขาด (agent ค้างถ้าไม่มี)
2. `2>&1 &` สำหรับ parallel + `wait $PID` สำหรับ sync
3. Brief file ครบ — agent ไม่มี memory ข้าม session

---

### Step 5: Final QA gate

หลัง implementation agents เสร็จ:
- ถ้ายังไม่ได้รัน Sage → รัน Sage เสมอ (`TOOLS_RO`)
- Sage approval = required ก่อน complete

---

### Step 6: Report to user

```markdown
## Task Complete: [Task Title]

**Status:** ✓ Complete
**Agents:** [list]
**Mode:** [sequential/parallel/mixed]

### What was done:
- Kai: [summary]
- Nova: [summary]
- Sage: [verdict]

### Files changed:
- [list]
```

## Routing Guide

| Task Type | Round 1 | Round 2 | Round 3 |
|-----------|---------|---------|---------|
| New API endpoint | Kai | — | Sage |
| New UI component | — | Nova | Sage |
| Full-stack feature | Kai | Nova ∥ Sage pre-check | Sage final |
| Bug fix (BE) | Kai | — | Sage |
| Bug fix (FE) | — | Nova | Sage |
| Auth/flow | Kai | Nova | Sage |
| Security audit | — | — | Sage |
| DB migration | Kai | — | Sage |

`∥` = parallel execution

---

## Example

```
/assign Add user profile page with avatar upload
```

ARIA decomposes:
- **Round 1 (Kai):** `POST /api/users/:id/avatar` + storage
- **Round 2 (Nova ∥ Sage):** Profile UI + upload component (parallel with Sage pre-review)
- **Round 3 (Sage):** Final integration test + file upload security check
